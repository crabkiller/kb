1. 2021年4月 - 2022年7月：搭建数字资产管理系统（DAM），提供可组装、可定制化的 SAAS 解决方案。前端采用 UIComponent（UI 层）、SDK（业务逻辑层）、Widget（组装层） 三层架构设计，既支持对 UI 和业务逻辑自由定制组装，同时也提供打包好的标准挂件方便一键接入使用。产出 UIComponent 30+、SDK 20+、Widget 10+。

2. 2022年8月 - 2022年12月：稿定设计主站重构升级，采用 DAM 相似三层架构，对视觉进行升级，梳理重构业务逻辑，提升性能和可维护性。

3. 2023年1月 - 2023年10月：稿定设计主站全面 SSR 化，在满足 SEO 诉求提升自然流量增长基础上，优化网站整体性能评分。整体评分从 60 提升至 85+
    - 响应式图像：根据不同 DPR、屏幕尺寸、webp 支持与否等条件，加载合适尺寸图片。LCP 降低 600+ms
    - 组件级懒渲染：将首屏不可见组件不渲染，抽象懒渲染机制在合适时机（hover/visible/delay）动态渲染，精简首屏 JS 和无用代码大小、加快水合。TBT/TTI 降低 200+ms
    - 瀑布流列表组件优化：由于瀑布流组件需要适配栅格系统，SSR 下无法获取屏幕宽度导致首屏渲染抖动，因此采用 script 标签动态插入瀑布流编排算法即时编排方式，解决视觉层面抖动问题。视觉体验极大增强、LCP 降低 500+ms
    - CDN 缓存页面：将 SSR 渲染结果缓存至 CDN 静态存储。命中缓存情况下 FCP 极大降低，600+ms
    - BFF 流程梳理优化：对 BFF 层冗余业务流程进行梳理剔除，将非必要串行流程并行化，FCP 降低 200+ms
    - 延迟水合：延迟非紧急交互区块水合时机，TBT/TTI 降低 300+ms

4. 稿定扩展能力设计开发，采用中介者模式解偶编辑器渲染和业务逻辑，将繁杂业务功能抽离为多个扩展动态插拔，降低项目复杂度、提升编辑器渲染层稳定性、提升整体加载和运行性能、便捷的业务功能上下线。
    - 可视化编辑器的核心架构设计：渲染引擎负责将各元素渲染到画布、插件系统负责用户在画布中对元素的操作交互、扩展系统提供更丰富的配置界面供用户编辑元素。
    - 扩展系统作为中介者，对画布的 CRUD 能力进行封装，提供给页面左侧栏、右侧栏、Toolbar 等各处的配置界面调用。

5. 2023年11月 - 2024年11月：稿定编辑能力整合，统一生产侧和用户侧编辑器，保持编辑能力一致性、降低研发成本；并采用 DAM 相似三层架构，对作图结果导出能力进行收拢，确保稿定各业务场景下作品导出能力一致性。
    - 将生产侧历史遗留旧版编辑器，替换为新版用户侧编辑器，模块级功能差异通过扩展挂载与否进行适配，细节点功能差异点抽象为权限配置进行适配
    - 作图结果导出依赖编辑器渲染引擎实例，在编辑器外其他业务场景下，不希望更不应该依赖编辑器渲染引擎，因此对渲染能力做最小化独立部署（preview 应用），外部业务方采用 iframe 方式加载 preview 应用完成出图
    - 出图能力拆分为 UI 组件（纯 UI 不依赖编辑器渲染引擎）、出图 SDK 能力（依赖编辑器渲染引擎实例），SDK 部署在 preview 应用中，外部业务方依赖 UI 组件包，并通过 postMessage 和 preview 应用交互获得 SDK 能力，拼装成 Widget 实现出图能力
